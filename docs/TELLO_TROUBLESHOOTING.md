# Tello SDK 常见问题排查表

## 连接问题

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 无法连接 Tello WiFi | WiFi 未开启/密码错误 | 确认连接 Tello 开头的 WiFi，无需密码 |
| 连接后无响应 | IP 地址错误 | 确认 UDP 目标 192.168.10.1:8889 |
| 频繁断连 | WiFi 干扰/距离过远 | 靠近 Tello（<3m），关闭其他 WiFi 设备 |
| command() 超时 | SDK 版本不兼容 | 使用 djitellopy >= 2.5.0 |

## 飞行控制问题

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 起飞后立即偏航 | IMU 未校准/地面不平 | 放在水平地面，重启后等待 5s 再起飞 |
| 移动指令无反应 | 未起飞/处于悬停模式 | 确认已起飞，速度指令范围 -100~100 |
| 移动距离不准 | 开环控制/风力影响 | 使用速度指令模式(模式B)，配合 EKF 闭环 |
| 悬停时漂移 | 光流地面纹理不足 | 确保地面有足够纹理（非纯白色） |

## 视频流问题

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 无视频画面 | 视频流未启动 | 调用 streamon() 后再获取帧 |
| 画面卡顿 | WiFi 带宽不足 | 降低分辨率到 480p，关闭其他网络应用 |
| 帧率低 | CPU 处理不过来 | 使用独立线程获取帧，主线程做检测 |
| 画面延迟大 | WiFi 延迟/缓冲 | 减少帧缓冲，使用最新帧而非队列 |

## 状态读取问题

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| get_height() 返回0 | 未起飞/气压计未稳定 | 起飞后等待 2s 再读取 |
| 姿态数据跳变 | 正常抖动/振动干扰 | 使用 EKF 滤波，增加过程噪声 |
| 电池百分比不准 | 电压估算误差 | 低于 20% 立即返航，不要冒险 |

## 紧急处理

```
任何异常 → 立即调用 emergency() 或 land()
严重漂移 → emergency() (电机立即停转，坠机)
一般异常 → land() (缓慢降落)
通信中断 → 自动悬停 → 尝试重连 → 超时自动降落
```

## 调试技巧

1. **先用 Mock 模式测试逻辑**
   ```python
   ctrl = TelloController(mode="mock")
   ```

2. **打印所有状态**
   ```python
   print(ctrl.get_state_dict())
   ```

3. **逐步增加高度**
   - 第一次真机测试: 目标高度 50cm
   - 确认稳定后: 逐步增加到 1m, 2m, 5m

4. **室内优先**
   - 无风环境验证控制器参数
   - 确认后再到室外测试
