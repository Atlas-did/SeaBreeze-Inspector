# PID / EKF 调参指南

> 本文档提供简化版 Ziegler-Nichols 调参法和 EKF 参数调整建议。

---

## 一、PID 调参 — Ziegler-Nichols 简化法

### 1. 什么是 PID？

PID 控制器根据"误差"（目标值 - 当前值）计算控制量：

```
输出 = Kp * 误差 + Ki * 误差积分 + Kd * 误差变化率
```

| 参数 | 作用 | 调大了会怎样 | 调小了会怎样 |
|------|------|-------------|-------------|
| Kp (比例) | 误差越大，输出越强 | 响应快但易振荡 | 响应慢，迟迟到不了目标 |
| Ki (积分) | 消除稳态误差 | 超调大，积分饱和 | 始终差一点点到不了 |
| Kd (微分) | 预测误差趋势，抑制超调 | 对噪声敏感，抖动 | 超调大，振荡 |

### 2. Ziegler-Nichols 调参步骤

**Step 1: 设 Ki=0, Kd=0, 从小到大调 Kp**

从 Kp=0.5 开始，每次翻倍，观察阶跃响应：
- Kp 太小：响应缓慢，需要很久才到目标
- Kp 合适：快速响应，略有超调
- Kp 太大：剧烈振荡，系统不稳定

记录出现**等幅振荡**时的 Kp 值，记为 Ku（临界增益）。

**Step 2: 测量振荡周期 Tu**

用秒表或代码记录连续两个波峰的时间差。

**Step 3: 按 Z-N 公式计算参数**

| 控制器类型 | Kp | Ki | Kd |
|-----------|-----|-----|-----|
| P | 0.5 * Ku | 0 | 0 |
| PI | 0.45 * Ku | 1.2*Kp/Tu | 0 |
| **PID** | **0.6 * Ku** | **2*Kp/Tu** | **Kp*Tu/8** |

> 示例：Ku=4.0, Tu=2.5s → Kp=2.4, Ki=1.92, Kd=0.75

### 3. 微调口诀

```
响应慢 ↑Kp，振荡大 ↓Kp
有静差 ↑Ki，超调大 ↓Ki
震荡猛 ↑Kd，噪声多 ↓Kd
```

### 4. 本项目的推荐参数（从仿真测试得出）

| 模式 | Kp | Ki | Kd | Kff | 说明 |
|------|-----|-----|-----|-----|------|
| 仿真测试 | 2.0 | 0.1 | 1.0 | -1.0 | 理想环境 |
| 室内真机 | 1.5 | 0.05 | 0.8 | -1.0 | WiFi延迟补偿 |
| 室外抗风 | 3.0 | 0.2 | 1.5 | -1.5 | 扰动较大 |

> 前馈增益 Kff 为负值：扰动向左，补偿向右。

## 二、EKF 调参

### 1. EKF 两个核心矩阵

| 矩阵 | 符号 | 作用 | 调大效果 | 调小效果 |
|------|------|------|---------|---------|
| 过程噪声 Q | 12x12 | 信任预测模型程度 | 更信任观测，响应快但抖 | 更信任模型，平滑但滞后 |
| 观测噪声 R | 6x6 | 信任传感器程度 | 更信任预测，平滑 | 更信任观测，响应快 |

### 2. 传感器特性参考

| 传感器 | 噪声水平 (R) | 延迟 | 备注 |
|--------|-------------|------|------|
| IMU 加速度 | 0.1-0.5 m/s^2 | <10ms | 短期准，长期飘 |
| 光学流 x,y | 1-5 cm | ~50ms | 低空有效 |
| 气压计高度 | 10-30 cm | ~100ms | 受风影响 |

### 3. 推荐 Q/R 比值

```python
# 正常悬停 (信任IMU)
Q_scale = 1.0
R_scale = 1.0

# 剧烈扰动 (更信任观测)
Q_scale = 0.5  # 减小Q = 更信任观测
R_scale = 1.0

# 传感器噪声大 (更信任模型)
Q_scale = 2.0
R_scale = 2.0
```

### 4. 自适应 Q

本项目已实现自适应 Q：当 Mahalanobis 距离 > 阈值时，自动增大 Q 以快速响应突变。

```python
# 在 disturbance_observer.py 中调整
self.mahalanobis_threshold = 5.0  # 默认值
# 扰动频繁的环境可降到 3.0
# 安静环境可提高到 8.0
```

## 三、联合调试流程

```
1. 先用仿真测试参数
   python backend/main.py --mode simulation

2. 单独测试 EKF
   python tests/test_ekf.py
   # 确认扰动估计误差 < 5%

3. 单独测试控制器
   python tests/test_controller.py
   # 确认阶跃响应收敛

4. 联合测试 EKF + 控制器
   python tests/test_integration.py

5. 真机测试（室内先）
   python backend/main.py --mode hardware
   # 先低空 (0.5m)，逐步升高
```

## 四、常见问题

**Q: 飞机一直晃怎么办？**
A: ↓Kp 或 ↑Kd，检查 WiFi 延迟是否过大。

**Q: 到不了目标点，差几厘米？**
A: ↑Ki，检查死区设置（默认 2cm）。

**Q: EKF 扰动估计跳来跳去？**
A: 增大 Q_scale，或提高 mahalanobis_threshold。

**Q: 风大的时候控制不住？**
A: ↑Kff（如 -1.5），确认 EKF 扰动估计正常输出。
